Welcome to the Causal Agent App

This application is designed to help you estimate causal effects from observational and A/B testing data using advanced statistical methods. You may also receive help from our AI Assistant in the dashboard.

1. Data Preparation

Simulated Data: If you don't have a dataset, select "Simulated Data" to explore the app's features. We provide two distinct sets of simulated data: one for Observational Analysis and one for Quasi-Experimental Analysis.

Upload Data: Upload your own CSV file. Ensure your data contains:

  • Treatment Column: The variable indicating the intervention (e.g., Feature_Adoption, Marketing_Campaign).
  • Outcome Column: The metric you want to influence (e.g., Account_Value, Conversion_Rate).
  • Confounders: Variables that influence both treatment and outcome (e.g., Customer_Segment, Region).

Data Preprocessing: Auto-Convert Booleans, Missing Value Imputation, Winsorization, Log Transformation, Standardization, Variable Bucketing, Data Filtering.

Data Quality & Readiness: Generate an AI summary and view the Mathematical Readiness Assessment (Traffic Light system) to see exactly which causal models your dataset supports.

Define Global Objective (Optional): Select your Target, Treatment, and Analysis Paradigm upfront. These globals automatically pre-configure all downstream charts and methods. For Spatiotemporal experiments (like GeoLift), your treatment is often an event space, so you may select "None" as the Treatment column.

2. Step-by-Step Causal Analysis

Select the analysis type based on your data structure. Observational for standard cross-sectional data, and Quasi-Experimental for time-series / natural experiments.

A. Observational Analysis

Use this for standard cross-sectional analysis or when you have user-level data without a time-series dimension.

Linear/Logistic Regression OLS / Logit

Use Case: Simple observational studies or standard A/B tests with few control variables.

Input: Select Treatment & Outcome. Add Confounders. Logit is automatically used for binary outcomes.

Result: ATE, P-value & Confidence Intervals. A p-value < 0.05 means the effect is statistically significant.

Matching & Weighting PSM / IPTW

Use Case: When treatment/control groups are "unbalanced" (e.g., test users are older than control).

Input: Requires Binary Treatment & Outcome. Add Confounders to match/re-weight.

Result: Adjusted ATE, P-value & Confidence Intervals.

Double Machine Learning DML

Use Case: For high-dimensional data or finding "Who" the treatment works best for (HTE).

Input: Select Treatment, Outcome, & Confounders (used as Effect Modifiers).

Result: ATE & CATE (HTE). View the Effect Modification table for segment effects.

Meta-Learners S/T-Learner

Use Case: Advanced ML approach to estimate Individual Treatment Effects (ITE) and segmentation.

Input: Select Treatment, Outcome, & Confounders (used as Effect Modifiers).

Result: ITE / CATE. Useful for Personalization.

B. Quasi-Experimental Analysis

Use this when you have time-series data (Pre/Post periods) or natural experiments.

Difference-in-Differences DiD

Use Case: Measuring impact when you have a Control Group and Pre/Post Periods.

Input: Select Treatment, Outcome, Time Period (0=Pre, 1=Post), and Control Variables. Logit option for binary outcome.

Result: DiD Estimation Results (Interaction Coefficient & P-value).

CausalImpact ITS / BSTS

Use Case: Measuring impact on a time series. Supports Panel Data, Filtered Unit, or aggregated data.

Input: Select Date, Outcome, Intervention Date. Add Control Variables or specify a Unit Col for panel/synthetic control mode.

Result: BSTS results showing credible intervals and actual vs. predicted plots.

GeoLift Synthetic Control

Use Case: Geographic split-testing optimized for market-level A/B testing.

Input: Select Date, Geography, Outcome, and Covariates. Modes: Power Analysis (Selection) & Impact Estimation (Post-Test).

Result: Market Selection/Power Curves or Post-Test Impact Estimation Scorecards.

CausalPy Bayesian Synthetic Control

Use Case: Pure-Python Bayesian alternative to GeoLift. Provides proper two-sided HDI credible intervals and posterior probabilities via PyMC.

Input: Select Date, Geography, Outcome (KPI), Treated Geography, and Intervention Date. Configure effect direction and HDI width.

Result: ATT, Cumulative Impact, Relative Lift with Bayesian HDI intervals and posterior probability of effect.
