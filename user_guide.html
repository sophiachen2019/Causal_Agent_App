Welcome to the Causal Agent App

This application is designed to help you estimate causal effects from observational and A/B testing data using advanced statistical methods. You may also receive help from our AI Assistant in the dashboard.

1. Data Preparation

Simulated Data: If you don't have a dataset, select "Simulated Data" to explore the app's features. We provide two distinct sets of simulated data: one for Observational Analysis and one for Quasi-Experimental Analysis.

Upload Data: Upload your own CSV file. Ensure your data contains:

  • Treatment Column: The variable indicating the intervention (e.g., Feature_Adoption, Marketing_Campaign).
  • Outcome Column: The metric you want to influence (e.g., Account_Value, Conversion_Rate).
  • Confounders: Variables that influence both treatment and outcome (e.g., Customer_Segment, Region).

Data Preprocessing features: Auto-Convert Booleans, Missing Value Imputation, Winsorization, Log Transformation, Standardization, Variable Bucketing, Data Filtering.

2. Step-by-Step Causal Analysis

Select the analysis type based on your data structure. Observational for standard cross-sectional data, and Quasi-Experimental for time-series / natural experiments.

A. Observational Analysis

Use this for standard cross-sectional analysis or when you have user-level data without a time-series dimension.

Linear/Logistic Regression OLS / Logit

Use Case: Simple observational studies or standard A/B tests with few control variables.

Input: Cross-sectional. Select Treatment & Outcome. Add Confounders to control for bias.

Result: ATE & P-value. A p-value < 0.05 means the effect is statistically significant.

Matching & Weighting PSM / IPTW

Use Case: When treatment/control groups are "unbalanced" (e.g., test users are older than control).

Input: Requires Binary Treatment. Uses Confounders to match or re-weight.

Result: Adjusted ATE. Focus on the 95% Confidence Interval.

Double Machine Learning DML

Use Case: For high-dimensional data or finding "Who" the treatment works best for (HTE).

Input: Uses ML to remove confounding noise from treatment and outcome.

Result: CATE & HTE Results. View the Effect Modification table for segment effects.

Meta-Learners S/T-Learner

Use Case: Advanced ML approach to estimate Individual Treatment Effects (ITE) and segmentation.

Input: Highly flexible model learning from treatment.

Result: ITE & Segment Analysis. Useful for Personalization.

B. Quasi-Experimental Analysis

Use this when you have time-series data (Pre/Post periods) or natural experiments.

Difference-in-Differences DiD

Use Case: Measuring impact when you have a Control Group and Pre/Post Periods.

Input: Long format data. Select Treatment, Outcome, and Time Period.

Result: View Interaction Coefficient and P-value.

CausalImpact ITS / BSTS

Use Case: Measuring impact on a time series. Supports Panel Data, Filtered Unit, or aggregated data.

Input: Select Date, Outcome, and Intervention Date. Add covariates if available.

Result: Scorecard & Plots. Focus on the 95% Confidence Intervals.

GeoLift Synthetic Control

Use Case: Geographic split-testing optimized for market-level A/B testing.

Input: Panel Data (Date & Location).

Result: Statistical Summary Scorecard (Estimation) or Power Curves (Selection).
